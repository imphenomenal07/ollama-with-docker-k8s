Yes. Running a 3B model with the official Ollama Docker image on Kubernetes in AWS is straightforward. The main decision is whether you want **CPU-only** (cheap, slower) or **GPU-backed** (recommended for production). The official image is `ollama/ollama`. ([Ollama][1])

### Architecture

```text
AWS EC2
 └── Kubernetes Node
      └── Ollama Pod
           ├── ollama/ollama
           ├── Persistent Volume (/root/.ollama)
           └── Port 11434
```

For a 3B model such as:

* Llama 3.2 3B
* Qwen 2.5 3B
* Gemma 3 4B

A `g5.xlarge` EC2 instance (NVIDIA A10G 24GB) is more than sufficient. Even a CPU instance can run 3B models, though inference will be slower.

---

## 1. Create a Persistent Volume

Ollama stores models under `/root/.ollama`. Without persistence, models will be downloaded again after pod restarts. ([Reddit][2])

```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: ollama-pvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 50Gi
```

---

## 2. Deploy Ollama

### CPU Version

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: ollama
spec:
  replicas: 1
  selector:
    matchLabels:
      app: ollama
  template:
    metadata:
      labels:
        app: ollama
    spec:
      containers:
      - name: ollama
        image: ollama/ollama:latest
        ports:
        - containerPort: 11434
        volumeMounts:
        - mountPath: /root/.ollama
          name: ollama-storage
      volumes:
      - name: ollama-storage
        persistentVolumeClaim:
          claimName: ollama-pvc
```

The official image exposes Ollama on port `11434`. ([Ollama][1])

---

## 3. GPU Deployment (Recommended)

First install:

* NVIDIA drivers on EC2
* NVIDIA Container Toolkit
* NVIDIA Device Plugin for Kubernetes

GPU support is officially supported through NVIDIA container runtime. ([Ollama Documentation][3])

Then:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: ollama
spec:
  replicas: 1
  selector:
    matchLabels:
      app: ollama
  template:
    metadata:
      labels:
        app: ollama
    spec:
      containers:
      - name: ollama
        image: ollama/ollama:latest
        ports:
        - containerPort: 11434
        resources:
          limits:
            nvidia.com/gpu: 1
        volumeMounts:
        - mountPath: /root/.ollama
          name: ollama-storage
      volumes:
      - name: ollama-storage
        persistentVolumeClaim:
          claimName: ollama-pvc
```

---

## 4. Expose Service

```yaml
apiVersion: v1
kind: Service
metadata:
  name: ollama
spec:
  selector:
    app: ollama
  ports:
  - port: 11434
    targetPort: 11434
  type: LoadBalancer
```

AWS will provision an external load balancer.

---

## 5. Download the 3B Model

After deployment:

```bash
kubectl exec -it deploy/ollama -- bash
```

Pull a model:

```bash
ollama pull llama3.2:3b
```

or

```bash
ollama pull qwen2.5:3b
```

Ollama models are downloaded separately from the Docker image. The image only provides the runtime. ([GitHub][4])

## 6. Test

```bash
curl http://<service-ip>:11434/api/generate \
```

## Production Recommendations on AWS

| Component  | Recommendation               |
| ---------- | ---------------------------- |
| Kubernetes | EKS                          |
| Storage    | EBS CSI Driver               |
| Instance   | g5.xlarge or g6e.xlarge      |
| Model      | Llama 3.2 3B or Qwen 2.5 3B  |
| Ingress    | AWS Load Balancer Controller |
| Monitoring | Prometheus + Grafana         |
| Scaling    | HPA + Cluster Autoscaler     |
